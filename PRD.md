# PRD — 「글로컬 AI 동행」 특강 신청·관리 포털

> 경상국립대학교 글로컬대학30 사업 · 2026학년도 「모두의 AI를 위한 글로컬 AI 동행」 특강
> 문서버전 v1.0 · 작성기준일 2026-06-30 · 대상 독자: 개발자 / 운영 관리자

---

## 1. 개요 · 배경 · 목적

### 1.1 배경
경상국립대학교 「글로컬대학30」 사업의 일환으로 **2026학년도 「모두의 AI를 위한 글로컬 AI 동행」 특강**을 운영한다. 교직원·재학생·지역민을 대상으로 7월 22일부터 8월 7일까지 총 5회차에 걸쳐 진행되며, AI 활용 역량을 수준별(초급~고급)로 제공한다.

### 1.2 목적
특강의 **온라인 신청·조회·만족도조사**와 **관리자 운영(신청자 관리·설문 집계)**을 단일 웹 포털에서 처리한다. 본 문서는 개발 착수 전 요구사항을 확정하는 **PRD(제품요구사항정의서)**로, 첫 산출물이다.

### 1.3 운영 개요
- **신청기간**: 2026-07-01 ~ 각 회차 시작 2일 전 자동 마감
- **운영기간**: 2026-07-22 ~ 2026-08-07 (총 5회차)
- **장소**: 디지털 트랜스랩 / 온라인(Zoom, 3차)
- **총 모집인원**: 180명
- **대상**: 교직원, 재학생, 지역민

### 1.4 산출물 범위
- **본 PRD(`PRD.md`) 문서만**이 이번 산출물이다. 실제 구현은 후속 작업.
- 참조 기술 스택: **Next.js + Supabase(Postgres) + Tailwind CSS**
- 카카오톡 알림: **설계(스펙 정의)만** 포함, 실제 발송 API 연동은 후속 범위.

## 5. 신청 포털 — 기능 요구사항

### 5.1 소개 탭
- 프로그램 소개
- **강사 소개**: 이성원 · 박용규 · 강수진 · 임근석
- 특강을 통해 얻는 역량 안내
- **회차별 일정표** (부록 §13 표 반영)
- **신청마감 로직 안내**: 각 회차 시작 **2일 전 자동 마감**

### 5.2 신청 탭
신청서 입력 필드:

| 필드 | 형식 / 검증 |
|---|---|
| 워크숍명 | 드롭다운(회차 5종 중 선택) |
| 성명 | 텍스트(필수) |
| 소속 | 텍스트(필수) |
| 교번/직번/학번/생년월일 | 식별번호(필수) |
| 연락처 | 휴대폰 형식 검증(필수) |
| 이메일 | 이메일 형식 검증(필수) |

- **개인정보 수집·이용 동의** 안내문 + 체크박스(**필수**)
- **정원/마감 검증**: 정원 초과 또는 마감 회차는 신청 불가 처리
- **제출 버튼**: 크고 **GNU 포인트 컬러로 강조**

### 5.3 신청내역조회 탭
- **성명 + 연락처** 입력 → 관리자 DB와 연동 조회
- 일치하는 본인 신청 건의 **상태/회차 정보** 반환
- (보안) 성명+연락처가 일치할 때만 본인 건만 반환

### 5.4 만족도조사 탭
- **참여 프로그램**(필수)
- **인지경로**(6지선다)
- **5점 척도 5문항** (1~3번 및 기타 필수표기 반영)
- **개방형 1문항**
- 첨부 문항을 그대로 반영

## 6. 관리자 포털 — 기능 요구사항

### 6.1 관리자 인증 · 보안
- 관리자 포털(신청자 관리·만족도 설문결과 **두 탭 전체**)은 **권한 인증 통과 시에만 접근**
- **Supabase Auth 구글 OAuth 로그인**으로만 접근 허용
- **허용 이메일 allowlist** 기반 권한 부여 — 초기 관리자: `eros4424@gmail.com`
- 비허용 계정 로그인 또는 미인증 접근 시: 모든 관리자 라우트는 **로그인 페이지로 리다이렉트(접근 차단)**
- **Supabase RLS**로 데이터 접근까지 이중 통제

### 6.2 신청자 관리 탭
- **컬럼**: 프로그램명, 신청일, 프로그램 일시, 성명, 소속, 교번/직번/학번/생년월일, 연락처, 이메일, 상태, 이수처리, 관리자에 의한 신청
- **필터 / 검색 / 엑셀 내보내기**
- **카카오톡 연동 안내문자 자동발송 설정 UI** (§9 스펙 참조 — 발송 토글, 템플릿 선택, 대상 선택, 발송/예약 버튼, 발송 이력)

### 6.3 만족도 설문결과 탭
- 응답 **집계·시각화**
- **`LAWdata` DB 테이블**로 원천 데이터 저장/내보내기

## 8. 신청마감 · 정원 비즈니스 규칙

- **마감 규칙**: 각 회차 운영 일시의 **2일 전** 자동 마감 (`deadline = start_at − 2일`)
- **정원 규칙**: 회차별 `capacity` 도달 시 신청 차단(또는 대기 처리 정책 적용)
- **마감/정원초과 회차**: 신청 탭 드롭다운에서 비활성 또는 "마감" 표기

## 9. 카카오톡 연동 안내문자 자동발송 스펙 (설계 전용)

> 본 PRD는 아래 기능을 **설계(스펙) 수준**으로 정의한다. 카카오 채널 개설·템플릿 심사·발송키 연동 등 **실제 연동은 후속 구현 범위**다.

### 9.1 목적 / 개요
- 신청자 연락처(휴대폰)로 카카오 **알림톡(비즈니스 채널 기반)** 안내문자 자동 발송
- 미수신/실패 시 **SMS(LMS) 대체발송(fallback)** 옵션

### 9.2 연동 아키텍처
- 카카오 비즈메시지 알림톡 API(또는 NHN/솔라피 등 발송 대행사) ↔ 백엔드 발송 트리거
- 자동화는 **연결된 Make 시나리오**로 오케스트레이션 가능
  `Supabase 이벤트 → Make → 카카오 알림톡`

### 9.3 메시지(템플릿) 유형 3종
1. **신청결과 안내** — 신청 상태 확정 시(승인/대기/취소)
   변수: `{성명}{프로그램명}{회차}{상태}{일시}{장소}`
2. **프로그램별 참여 유의사항** — 회차별 준비물·주의(예: 노트북·Claude Pro 지참, 카메라/마이크 환경)
   변수: `{성명}{프로그램명}{유의사항}`
3. **프로그램 안내** — 장소·시간·간략 내용
   변수: `{성명}{프로그램명}{일시}{장소}{내용요약}{Zoom링크}`

### 9.4 발송 트리거
- (a) 신청완료 즉시 자동
- (b) 관리자 상태 변경 시 자동
- (c) 프로그램 D-N 예약발송(스케줄러/Make)
- (d) 관리자 수동 일괄발송(선택 신청자 대상)

### 9.5 관리자 발송 설정 UI
- 자동발송 ON/OFF 토글
- 템플릿별 활성화
- 예약 시점 설정
- 대상 필터(회차·상태)
- 발송 미리보기
- **발송 이력/결과**(성공·실패·재발송)

### 9.6 데이터
- `kakao_notifications` 테이블(설계용) — §7.5 참조

### 9.7 개인정보 / 동의
- 안내 발송 목적의 연락처 활용 동의(신청서 정보수집동의에 포함)
- 광고성 여부 구분(**정보성 메시지**)
